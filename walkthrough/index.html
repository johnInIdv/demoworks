A 2-minute walkthrough

Pitchcraft

A folder-based specialist that turns a customer discovery-call transcript into a tailored, working product demo.

01 — The problem

Every prospect demo is hand-built from the call.

3–5 hrs
to build each custom demo by hand
Generic
templates land flat — a brochure, not their workflow
Zero
compounding — every transcript starts from scratch

The judgment — which pain maps to which capability — lives only in my head.

02 — The fix

So I built the folder I wished I had.

Drop a transcript in — it does what I do, in order, and produces a demo.

pitchcraft/
├─ brief.md — the real problem
├─ identity.md · rules.md — who it is + the method
├─ reference/ — the knowledge (the leverage)
└─ examples/ — a full worked demo
03 — Where the leverage lives

The method is generic. The knowledge makes it yours.

reference/
├─ solution-patterns.md — customer-pain → capability playbook
├─ product-capabilities.md — what it does + honest limits
└─ triage-criteria.md — what's worth a deep build

Swap reference/ and you re-point Pitchcraft at a different product — without touching the method.

05 — The trigger

Paste a transcript. That's the whole action.

pitchcraft % claude
▸ reads CLAUDE.md — "a transcript is the go"
> [ paste discovery-call transcript ]
▶ Stage 01 Triage… 02 Workflow… 03 Opportunity map… → preview.html

No prompt, no setup. Claude runs the pipeline and writes every stage.
